crossSectionAdjustForHeatedTarget Module¶

crossSectionAdjustForHeatedTarget.lib.crossSectionAdjustForHeatedTarget.crossSectionAdjustForHeatedTarget()¶

crossSectionAdjustForHeatedTarget( massRatio, T, crossSection, lowerlimit = ‘constant’, upperlimit = ‘constant’, interpolationAccuracy = 0.002, heatAllPoints = 0, doNotThin = 0, EMin = 1e-11 ) Returns the cross-section adjusted for a target with temperature T. The unit of T must be the same as for the energy data. massRatio is the ratio of the target’s mass to the projectile’s mass. lowerlimit and upperlimit specify how the input data is extended beyond its energy range when performing integrations over the input energy range. Valid strings for lowerlimit are ‘constant’, ‘oneOverV’ and ‘threshold’. Valid strings for upperlimit are ‘constant’ and ‘oneOverV’. The heated data is thicken to have an interpolation accuracy of interpolationAccuracy. If interpolationAccuracy is greater [less] than 0.1 [1e-6], it is set to .1 [1e-6]. crossSection must be a list of [ E, xsec ] numbers. For example crossSection = [ [ 1e-10, 2 ], [ 20, 2. ] ] would represent a constant cross-section of value 2 from energy 1e-10 to 20. If heatAllPoints is true than all points are heated otherwise, a judicial choice of E points is made. This latter procedure, which is recommended, is typically much faster. If doNotThin is true then points are not thinned, otherwise thinning, which is recommended, is performed. For threshold data, energy values below threshold are generated. EMin is the minimum energy that shall be generater. For energy in MeV, the default is appropriated, otherwise, a compariable value in the energy unit should be entered.
