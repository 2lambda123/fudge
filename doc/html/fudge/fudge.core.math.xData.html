class fudge.core.math.xData.LegendreSeries.XYs_LegendreSeries(coefficients, index=None, value=None, parent=None)[source]¶

Bases: fudge.core.ancestry.ancestry

This class stores and manipulates an angular pdf (i.e. pdf(mu) where mu is the cos of the angle) represented as Legendre coefficients. The pdf and Legendre coefficients are related by

pdf(mu) = Sum_over_l_of ( l + 0.5 ) * C_l * P_l(mu)

where the sum is from l = 0 to lMax, lMax is the highest Legendre coefficients in the instance, C_l is the Legendre coefficient for Legendre order l and P_l(mu) is the Legendre polynomial of order l.

XYs Module¶

This module contains the class XYs. This class treats a list of \((x_i, y_i)\) pairs as if they were the function \(y(x)\). That is, it is a numerical representation of \(f(x)\). As an example, let

\[y_1(x) = a + b * x\]

and

\[y_2(x) = c * \exp( - d * x )\]

then, \(y_1\) and \(y_2\) can be added, subtracted, multiplied and divided, for example. Similiarly, two XYs instances can be added, subtracted, multiplied and divided. The only restriction on the \((x_i, y_i)\) pairs is that \(x_i < x_{i+1}\).

The XYs class uses the class numericalFunctions.lib.pointwiseXY_C.pointwiseXY_C as a base class.

Members of an XYs instance of interest to most are:

accuracy:see the base class numericalFunctions.lib.pointwiseXY_C.pointwiseXY_C,
biSectionMax:see the base class numericalFunctions.lib.pointwiseXY_C.pointwiseXY_C,
infill:see the base class numericalFunctions.lib.pointwiseXY_C.pointwiseXY_C,
axes:Description of the x and y data attributes (e.g., label, interpolation, units).

A XYs object can be the XY instances of higher dimensional math objects like W_XYs. In this case, the XYs instances are secondary instances.

_group(xs, f2=None, f3=None, norm=None, asXYs=False)[source]¶

This function will be renamed to group when it is completed. Called _group for now so that it does not conflict with Dave Brown’s version when using svn. That is, this version will eventually replace Dave’s.

The argument xs is a list of x-values with xs[i] < xs[i+1]. This function calculates the integrals

\[\int_{xs[i]}^{xs[i+1]} dx \; { f(x) \over n[i] }\]
where \(f(x)\) is defined as
\(f(x)\) conditional
\(f_1(x)\) for f2 = f3 = None
\(f_1(x) \, f_2(x)\) for f3 = None
\(f_1(x) \, f_3(x)\) for f2 = None
\(f_1(x) \, f_2(x) \, f_3(x)\) otherwise
\(f_1(x)\) is self evaluated at x and n[i] is a normalization determined from the norm arugment as
norm n[i]
None 1
‘dx’ x[i+1] - x[i]
a python list norm[i]

The arguments f2 (and f3) must be None of an XYs instance.

If asXYs is False, then len( xs ) - 1 integrals are returned. If asXYs is True, the last integral’s values is appended to the end to make a list of length len( xs ), and an instance of class XYs is returned with the x-values from xs, the y-values from the integrals and the interpolation is ‘linear,flat’.

Historical note: the word group comes from deterministic neutron transport (e.g., transport used to simulate nuclear reactors.

applyFunction(f, parameters, accuracy=-1, biSectionMax=-1, checkForRoots=False)[source]¶

applyFunction maps the function ‘f’ onto each y-value in an XYs object, returning a new XYs object. Additional points may be added to preserve the desired accuracy.

For example, the following will transform all negative y-values in an XYs object to zero: >>> newXYs = XYs.applyFunction( lambda y,tmp: 0 if y<0 else y, None )

Extra parameters to the applied function can be passed in the ‘parameters’ argument. See XYs.XYs documentation for details on ‘accuracy’ and ‘biSectionMax’

toPointwise_withLinearXYs(accuracy, lowerEps, upperEps, removeOverAdjustedPoints=False, axes_=None)[source]¶

Converts the regions of self into a single XYs.XYs instance that has ‘linear,linear’ interpolation. At the boundary between two abutting regions, the x-values are the same, which is not allowed for an XYs.XYs instance. The arguments lowerEps, upperEps are used to smear the x-values at a boundary as follows. Let \((x_l, y_l)\) and \((x_u, y_u)\) be the abutting points for two abutting regions. If \(y_l = y_u\) then the point \((x_u, y_u)\) is removed. Otherwise, if( lowerEps > 0 ) the point \((x_l, y_l)\) is moved to \(x = x_l * ( 1 - lowerEps )\) (or \(x = x_l * ( 1 + lowerEps )\) if \(x_l < 0\)) and the \(y\) value is interpolated at \(x\). If \(x\) is less than the x-value of the point below \((x_l, y_l)\) and removeOverAdjustedPoints is True then the point \((x_l, y_l)\) is removed; otherwise, a raise is executed. Similarly for upperEps and the point \((x_u, y_u)\).
